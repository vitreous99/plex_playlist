# Phase 3 — Sonic Fulfillment & Playback

**Goal:** Close the loop from LLM suggestions to actual Plex tracks, enrich playlists with Plex's sonic analysis, and push playback to remote clients.

**Prerequisite:** Phase 2 complete (metadata synced, LLM returns structured suggestions via `/api/suggest`).

---

| # | Step | Description | Definition of Done |
|---|------|-------------|-------------------|
| 3.1 | **Build the Track Matcher** | Create `backend/app/services/track_matcher.py`. Given a `PlaylistResponse`, for each `SuggestedTrack`: (a) search the SQLite cache by title + artist (fuzzy-tolerant), (b) if found, fetch the real `plexapi.audio.Track` object by `rating_key`, (c) if not found, log it as unmatched. Return `(matched: list[Track], unmatched: list[SuggestedTrack])`. | Given LLM suggestions against a synced library, matched items are real Plex Track objects; unmatched items are logged with title and artist. |
| 3.2 | **Implement Sonic Similarity expansion** | Create `backend/app/services/sonic_engine.py`. Given a list of matched Track objects and a target playlist length: call `track.sonicallySimilar(limit=50, maxDistance=0.25)` on each seed track. Collect results, de-duplicate by `ratingKey`, and trim to the target length. | A 5-seed input expands to a 25-track playlist of sonically related, de-duplicated songs. |
| 3.3 | **Implement Sonic Adventure for transition playlists** | Extend `sonic_engine.py` with `build_sonic_adventure(source_track, target_track)`. Detect transition-style prompts (e.g., "start with Folk, end with Metal"). Use `sonicAdventure(to=target_track)` to build an acoustic path. | Given a source and destination track, returns an ordered list of tracks bridging them sonically. |
| 3.4 | **Build the Playlist Assembly pipeline** | Create `backend/app/services/playlist_builder.py`. Orchestrate the full flow: receive prompt → call `/api/suggest` logic → Track Matching → Sonic Expansion (or Sonic Adventure if transition detected) → return final ordered list of Track objects. Expose as `build_playlist(prompt: str, track_count: int) -> list[Track]`. | `build_playlist("mellow indie vibes", 30)` returns a list of ≈30 real Plex Track objects. |
| 3.5 | **Implement Client Discovery** | Create `backend/app/services/client_dispatcher.py`. Call `plex.clients()` to list available playback clients. Return a list of `{name, identifier, product, address}`. If a specific client is unreachable, attempt `client.proxyThroughServer()`. Add `GET /api/clients` endpoint. | `GET /api/clients` returns a JSON list of available Plex clients with their details. |
| 3.6 | **Implement PlayQueue creation & playback dispatch** | Extend `client_dispatcher.py` with `dispatch_playback(tracks, client_name)`. Create a `PlayQueue` via `PlayQueue.create(server, items=tracks)`, find the client by name, and call `client.playMedia(playqueue)`. Handle errors: client not found, client asleep, playback failure. | Calling the function with a track list and client name starts playback on the target device. |
| 3.7 | **Expose end-to-end playlist + play endpoint** | Add `POST /api/playlist/play` accepting `{prompt: str, track_count: int, client_name: str?}`. Orchestrate the full pipeline: build_playlist → dispatch_playback. Return `{playlist_name, track_count, tracks: [{title, artist}], client, status}`. | A single API call with a natural-language prompt results in music playing on the selected client. |
| 3.8 | **Add "save as permanent playlist" endpoint** | Add `POST /api/playlist/save` accepting `{prompt: str, track_count: int, playlist_name: str?}`. Build the playlist, then call `Playlist.create(server, title=name, items=tracks)`. Return the playlist name and track count. | The created playlist appears in the Plex UI under Playlists. |
| 3.9 | **Error handling & resilience hardening** | Audit all Phase 3 services for failure modes: Plex connection drops mid-operation, Ollama timeouts, zero sonic results (track has no analysis), no available clients, Shield TV asleep. Return structured HTTP error responses (4xx/5xx) with descriptive messages. Add `GET /api/diagnostics` endpoint checking Plex connectivity, Ollama availability, DB sync freshness, and GPU status. | Every failure mode returns a clear, actionable JSON error; `/api/diagnostics` gives a full system health report. |
